Presentation av Kim Nilsson UD14

Preprocessor

Skriptspråk som utökar CSS språket.

Varför ska man använda preprocessors?

Renare, lättläst och återanvändbart.

Flexibilitet, exempelvis "om detta, gör det här".

Snuttar av kod och bibliotek som går att dela med andra.

Cross browser compability.

Vad är då The Semantic Grid System?

Tillägg till en preprocessor.

Float och dylikt ersätts av X antal kolumner och rader.

Detta skrivs med LESS:

@import 'grid.less'; @columns: 12; @column-width: 60; @gutter-width: 20; article { .column(9); } section { .column(3); }

Såhär översätts det till CSS:

article { display: inline; float: left; margin: 0px 10px; width: 700px; } section { display: inline; float: left; margin: 0px 10px; width: 220px; }

Fler saker LESS kan göra...

.color: blue; #section{ color: @color; }

Exemplet sätter textinnehållet i section till blå.

Man kan också använda sig av plus, minus, multiplikation osv.

.border-width: 1px; #nav{ border: @border-width*3; }

Mixins

.rounded(@radius) { -webkit-border-radius: @radius; -moz-border-radius: @radius; border-radius: @radius; } #header { .rounded(5px); }

I CSS blir det såhär:

#header { -webkit-border-radius: 5px; -moz-border-radius: 5px; border-radius: 5px; }
